Breadth-First Search (BFS)

Algorithm Outline

  • Always select from the OPEN list the node with the smallest depth (shallowest node) for expansion.
  • Put all newly generated nodes into the OPEN list.
  • OPEN is organized as a FIFO (first-in, first-out) list, i.e., a queue.
  • Terminate if a node selected for expansion is a goal.

Example

S CBA D GE 1 5 8 9 4 5 3 7

BFS Execution

  • Initial State:

    • OPEN list: { S }
    • CLOSED list: { }

  • Step-by-Step Expansion:

    Expanded Node OPEN List CLOSED List
    S { A B C } {S}
    A { B C D E G } {S A}
    B { C D E G G' } {S A B}
    C { D E G G' G" } {S A B C}
    D { E G G' G" } {S A B C D}
    E { G G' G" } {S A B C D E}
    G { G' G" } {S A B C D E}

  • Solution Path: S -> A -> G (This G also has a cost of 10)

  • Number of Nodes Expanded: 7

Evaluating Search Strategies

  • Completeness: BFS is complete if the branching factor is finite; it will find a solution if one exists.
  • Time Complexity: O(b^d), where b is the branching factor and d is the depth of the shallowest goal node.
  • Space Complexity: O(b^d), as it needs to store all generated nodes in the frontier.
  • Optimality: BFS is optimal if all path costs are the same, as it finds the shallowest solution first.

Example Calculation

  • For a complete search tree of depth d, where each non-leaf node has b children:

    • Total Nodes: 1 + b + b^2 + ... + b^d = (b^(d+1) - 1)/(b-1)
    • For a tree of depth 12 with b=10, total nodes: O(10^12)

  • BFS is suitable for problems with shallow solutions.

Depth-First Search (DFS)

Algorithm Outline

  • Always select from the OPEN list the node with the greatest depth for expansion.
  • Put all newly generated nodes into the OPEN list.
  • OPEN is organized as a LIFO (last-in, first-out) list, i.e., a stack.
  • Terminate if a node selected for expansion is a goal.

Example

S CBA D E 1 5 93 7 G G’ G” 8 54

UCS Execution

  • Initial State:

    • OPEN list: { S(0) }
    • CLOSED list: { }

  • Step-by-Step Expansion:

    Expanded Node OPEN List CLOSED List
    S { A(1) B(5) C(8) } {S}
    A { D(4) B(5) C(8) E(8) G(10) } {S A}
    D { B(5) C(8) E(8) G(10) } {S A D}
    B { C(8) E(8) G’(9) G(10) } {S A D B}
    C { E(8) G’(9) G(10) G”(13) } {S A D B C}
    E { G’(9) G(10) G”(13) } {S A D B C E}
    G’ { G(10) G”(13) } {S A D B C E G’}

  • Solution Path: S -> B -> G (This G has a cost of 9, not 10)

  • Number of Nodes Expanded: 7

Evaluating Search Strategies

  • Completeness: UCS is complete, as long as the cost of each step is greater than some small positive constant.
  • Time Complexity: O(b^d), similar to BFS in the worst case but depends on the path costs.
  • Space Complexity: O(b^d), since it keeps all generated nodes in memory.
  • Optimality: UCS is optimal if path costs are non-negative because it expands the least costly path first.

Worst Case Scenario

  • UCS becomes BFS when all arcs cost the same.

Depth-Limited Search (DLS)

Overview

  • Depth-Limited Search is a modified version of DFS that imposes a limit on the depth of the search.
  • It explores nodes up to a certain depth, preventing excessive deep paths that are unlikely to lead to the goal.

Evaluating DLS

  • Completeness: DLS is complete if the depth limit is greater than or equal to the depth of the shallowest goal. If the limit is too low, it may miss the solution.
  • Time Complexity: O(b^l), where l is the depth limit.
  • Space Complexity: O(b^l), as it needs to store up to l nodes at the deepest level.
  • Optimality: DLS is not optimal, as it might not find the least-cost solution if the limit is set too low.

Depth-First Iterative Deepening Search (DFID)

Overview

  • Combines the advantages of BFS and DFS.
  • Is complete, has linear space complexity, and has a time complexity of O(b^d).

Algorithm Outline

  1. Perform DFS to depth 0 (i.e., treat start node as having no successors).
  2. If no solution is found, perform DFS to depth 1.
  3. Continue incrementing depth and performing DFS until the solution is found.

Evaluating DFID

  • Completeness: DFID is complete as it incrementally increases the depth limit until it finds the goal.
  • Time Complexity: O(b^d), as it performs a DFS up to each level d.
  • Space Complexity: O(b^d), similar to DFS, as it only needs to store the current path from the root to the leaf node.

Bidirectional Search

Overview

  • Bidirectional Search performs two simultaneous searches: one forward from the start node and one backward from the goal node.
  • It uses two search trees that meet in the middle.

Evaluating Bidirectional Search

  • Completeness: Bidirectional Search is complete if the forward and backward searches meet, and if the branching factor is finite.
  • Time Complexity: O(b^(d/2)), as it searches from both directions.
  • Space Complexity: O(b^(d/2)), because it needs to store nodes in both search trees.

Greedy Best-First Search

Algorithm Outline

  • Selects nodes based on their heuristic value.
  • Always expands the node that appears to be closest to the goal according to the heuristic.

Evaluating Greedy Best-First Search

  • Completeness: Not complete, as it may get stuck in loops or fail to find a solution.
  • Time Complexity: O(b^m), where m is the maximum depth of the search tree.
  • Space Complexity: O(b^m), similar to DFS.
  • Optimality: Not optimal, as it does not consider path costs.

A* Search

Algorithm Outline

  • Combines the features of UCS and Greedy Best-First Search.
  • Uses both path cost g(n) and heuristic estimate h(n) to evaluate nodes: f(n) = g(n) + h(n).

Evaluating A* Search

  • Completeness: Complete if the heuristic function is admissible (i.e., never overestimates the cost to reach the goal).
  • Time Complexity: O(b^d), similar to UCS in terms of worst-case complexity.
  • Space Complexity: O(b^d), similar to UCS.

Hill Climbing

Overview

  • A local search algorithm that continuously moves towards the direction of increasing value (i.e., towards the goal).

Evaluating Hill Climbing

  • Completeness: Not complete, as it may get stuck in local maxima.
  • Time Complexity: O(b^m), where m is the maximum depth of the search tree.
  • Space Complexity: O(b^m), similar to DFS.
  • Optimality: Not optimal, as it may not find the global maximum.

Simulated Annealing

Overview

  • A probabilistic technique that explores the solution space by accepting less optimal solutions with a decreasing probability.

Evaluating Simulated Annealing

  • Completeness: Complete if the cooling schedule is appropriately chosen.
  • Time Complexity: Depends on the cooling schedule and stopping condition.
  • Space Complexity: Depends on the implementation.

Beam Search

Overview

  • A variation of BFS that only expands a limited number of nodes at each level.

Evaluating Beam Search

  • Completeness: Not complete if the beam width is too small.
  • Time Complexity: O(w^d), where w is the beam width.
  • Space Complexity: O(w^d), where w is the beam width.

Genetic Algorithm

Overview

  • An optimization algorithm inspired by the process of natural selection.
  • Uses operations such as selection, crossover, and mutation to evolve solutions.

Evaluating Genetic Algorithm

  • Completeness: Not always complete; depends on population size and evolutionary process.
  • Time Complexity: Depends on population size and number of generations.
  • Space Complexity: Depends on population size and representation of solutions.
