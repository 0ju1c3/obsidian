Intelligent (Rational) Agents Study Notes

1. Introduction to Agents and Environments

1.1 Definition of an Agent

  • An agent is anything that can perceive its environment through sensors and act upon that environment through actuators.

1.2 Types of Agents

  1. Human Agent

    • Sensors: eyes, ears, and other sensory organs
    • Actuators: hands, legs, mouth, and other body parts

  2. Robotic Agent

    • Sensors: cameras, infrared rangefinders, etc.
    • Actuators: various motors

  3. Software Agent

    • Example: A thermostat
      • Sensor: temperature detector
      • Actuator: heating/cooling system control

1.3 Agent Function and Program

  • Agent function: maps percept histories to actions [f: P* → A]
  • Agent program: runs on the physical architecture to produce f
  • Relationship: agent = architecture + program

2. Rationality in Agents

2.1 Definition of a Rational Agent

A rational agent selects actions that are expected to maximize its performance measure, given:

  1. The evidence provided by the percept sequence
  2. The agent's built-in knowledge

2.2 Factors Affecting Rationality

  1. Performance measure defining the criterion of success
  2. Agent's prior knowledge of the environment
  3. Actions that the agent can perform
  4. Agent's percept sequence to date

3. PEAS Framework

PEAS stands for Performance measure, Environment, Actuators, and Sensors. It's a framework for specifying task environments for intelligent agents.

3.1 Components of PEAS

  1. Performance measure: How we evaluate the agent's behavior
  2. Environment: The external world in which the agent operates
  3. Actuators: The means by which the agent can act on its environment
  4. Sensors: The means by which the agent perceives its environment

3.2 PEAS Examples

3.2.1 Autonomous Car

  • Performance measure: Safe driving, following traffic rules, passenger comfort, minimizing travel time
  • Environment: Roads, other vehicles, pedestrians, traffic signals, weather conditions
  • Actuators: Steering wheel, accelerator, brakes, signals
  • Sensors: Cameras, LIDAR, GPS, speedometer, thermometer

3.2.2 Playing Soccer

  • Performance: Goals scored, goals prevented, games won
  • Environment: Soccer field, other players, ball, weather conditions
  • Actuators: Legs for running and kicking, head for heading the ball
  • Sensors: Eyes for visual input, ears for auditory cues, tactile sensors for feeling the ball

3.2.3 Exploring Titan's Subsurface Oceans

  • Performance: Amount and quality of data collected, survival time
  • Environment: Subsurface oceans of Titan, extreme pressure and temperature
  • Actuators: Propulsion system, sampling tools, data transmission antenna
  • Sensors: Pressure sensors, temperature sensors, chemical analyzers, cameras

3.2.4 Shopping for Used AI Books Online

  • Performance: Book quality, price, shipping time
  • Environment: Online marketplaces, websites, databases
  • Actuators: Web browser interactions (clicks, form submissions)
  • Sensors: Text recognition for reading listings, price comparators

4. Environment Types

Environments can be categorized based on various properties:

  1. Fully Observable vs. Partially Observable

    • Fully Observable: Agent can obtain complete information about the environment's state (e.g., chess board)
    • Partially Observable: Agent has limited or noisy sensors (e.g., poker game)

  2. Deterministic vs. Stochastic

    • Deterministic: Next state is completely determined by current state and agent's action (e.g., puzzle-solving)
    • Stochastic: There's uncertainty in the outcome of actions (e.g., robot navigation in a crowded area)

  3. Episodic vs. Sequential

    • Episodic: Agent's experience is divided into atomic episodes (e.g., image classification)
    • Sequential: Current decisions affect future decisions (e.g., chess game)

  4. Static vs. Dynamic

    • Static: Environment doesn't change while agent is deliberating (e.g., crossword puzzle)
    • Dynamic: Environment changes over time, even while the agent is thinking (e.g., real-time strategy game)

  5. Discrete vs. Continuous

    • Discrete: Limited number of distinct percepts and actions (e.g., chess)
    • Continuous: Percepts and actions vary in a continuous range (e.g., driving a car)

  6. Single-agent vs. Multi-agent

    • Single-agent: Only one agent operates in the environment (e.g., puzzle-solving)
    • Multi-agent: Multiple agents interact in the environment (e.g., multiplayer games, economic markets)

4.1 Example: Chess Environment

  • Fully Observable: Both players can see the entire board
  • Deterministic: The next state is completely determined by the current state and the player's action
  • Sequential: Earlier moves affect future decisions
  • Static: The game state doesn't change while a player is thinking
  • Discrete: There are a finite number of possible moves at each turn
  • Two-agent: Two players compete against each other

5. Agent Types

  1. Simple Reflex Agents

    • Select actions based only on the current percept
    • Example: Thermostat that turns on heating when temperature drops below a threshold

  2. Model-Based Reflex Agents

    • Maintain internal state to track aspects of the world not visible in the current percept
    • Example: Autonomous car that keeps track of nearby vehicles even when they're not in direct view

  3. Goal-Based Agents

    • Work towards predefined goals
    • Example: Navigation systems that find routes to specified destinations

  4. Utility-Based Agents

    • Maximize a utility function that maps states to real numbers representing desirability
    • Example: Economic agent making investment decisions to maximize profit

  5. Learning Agents

    • Can improve their performance over time through experience
    • Example: Recommendation systems that learn user preferences over time

6. Vacuum-Cleaner World Example

6.1 Environment Description

  • Two locations: A and B
  • The vacuum cleaner can be in either location
  • Each location can be clean or dirty

6.2 PEAS Description

  • Performance measure: Amount of dirt cleaned up in a fixed time period
  • Environment: Two locations (A and B) which can be clean or dirty
  • Actuators: Left, Right, Suck, NoOp (No Operation)
  • Sensors: Location sensor, Dirt sensor

6.3 Simple Reflex Agent Program

def vacuum_agent(location, status):
    if status == "Dirty":
        return "Suck"
    elif location == "A":
        return "Right"
    elif location == "B":
        return "Left"

This simple program demonstrates how a reflex agent might operate in the vacuum-cleaner world, making decisions based solely on its current percepts.

Formalizing Search in a State Space

1. State Space Definition

A state space is a graph, (V, E) where:

  • V is a set of nodes
  • E is a set of arcs
  • Each arc is directed from one node to another node due to an action executed

2. Components of a State Space

2.1 Nodes

  • Correspond to a state
  • Include:
    • State description
    • Optionally:
      • Parent information
      • Operation to generate the node from its parent
      • Other bookkeeping data

2.2 Arcs

  • Correspond to an applicable action/operation
  • Properties:
    • Connect a source node (parent) to a destination node (child)
    • Have a fixed, non-negative cost associated with them

2.3 Relationships

  • Parent: Immediate predecessor of a node
  • Child: Immediate successor of a node
  • Ancestors: All predecessors of a node
  • Descendants: All successors of a node

3. Node Operations

3.1 Node Generation

  • Making a node explicit by applying an action to another explicit node

3.2 Node Expansion

  • Generating all children of an explicit node by applying all applicable operations

4. Search Components

4.1 Start Nodes

  • One or more nodes designated as starting points

4.2 Goal Test

  • A predicate applied to a node to determine if its associated state is a goal state

5. Solution

5.1 Definition

  • A sequence of operations associated with a path in the state space from a start node to a goal node

5.2 Cost

  • The sum of the arc costs on the solution path
