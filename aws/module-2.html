AWS Compute & Storage Services

Module 2

Introduction to AWS Compute

  • AWS Compute is a type of Infrastructure as a Service (IAAS). In simple terms, it allows users to provision and manage computing resources like virtual machines and containers for various use cases. These resources are essential for hosting applications or running computation-intensive tasks.

  • Key AWS Compute Services:

    • Amazon Elastic Compute Cloud (EC2):
      • EC2 allows users to launch and manage virtual servers (known as instances).
      • Examples: Running a web server, hosting a database, or even running complex machine learning models.
    • Amazon Elastic Container Service (ECS):
      • ECS is used to manage and run Docker containers in a highly scalable manner.
      • Example: Deploying a microservices architecture using Docker containers.
    • Amazon Elastic Kubernetes Service (EKS):
      • EKS simplifies running Kubernetes on AWS, managing clusters and automating critical tasks.
      • Example: Managing containerized applications with Kubernetes.
    • Amazon Lightsail:
      • Provides easy-to-use cloud services for smaller workloads, ideal for users who are not cloud experts.
      • Example: Launching a simple WordPress site or setting up a development environment.
    • AWS Lambda:
      • A serverless compute service where users can run code without provisioning or managing servers.
      • Example: Automatically resizing images when they are uploaded to an S3 bucket.

  • Benefits of AWS Compute:

    • Dynamic Provisioning:
      • Users can dynamically adjust the number of resources they are using. For instance, if a website experiences a sudden surge in traffic, additional EC2 instances can be automatically provisioned to handle the load.
    • Cost Efficiency:
      • With AWS Compute, users pay only for the resources they actually use. This pay-as-you-go model reduces the need for significant upfront capital investment, as seen in traditional IT infrastructures where companies had to purchase and maintain their own servers.
    • Management by AWS:
      • AWS takes care of all the underlying infrastructure management, including hardware maintenance, backups, emergency recovery, and ensuring enough server capacity during traffic surges. This allows companies to focus on their core business rather than IT infrastructure.

Amazon EC2 (Elastic Compute Cloud)

  • Overview:

    • Amazon EC2 provides scalable computing capacity in the AWS cloud. It is designed to make web-scale cloud computing easier for developers.
    • Key Features:
      • Security: EC2 instances are isolated from each other, providing a secure environment.
      • Resizability: Users can easily scale up or down their compute capacity based on their needs.
      • Flexibility: EC2 supports various operating systems and architectures, allowing users to choose the environment that best suits their applications.

  • Instance Types:

    • On-Demand Instances:

      • With On-Demand Instances, you pay for compute capacity by the hour or second without any long-term commitments.
      • Use Cases:
        • Ideal for short-term, spiky, or unpredictable workloads, such as launching a new marketing campaign that might suddenly increase web traffic.
        • Suitable for applications being developed or tested on Amazon EC2 for the first time.
      • Example:
        • A startup developing a new app might use On-Demand Instances to quickly scale their infrastructure as they iterate and test their application.

    • Spot Instances:

      • Spot Instances allow users to bid on spare Amazon EC2 computing capacity, which can result in savings of up to 90% compared to On-Demand pricing.
      • Use Cases:
        • Applications with flexible start and end times, such as big data analysis or batch processing jobs.
        • Scenarios where the application can tolerate interruptions, as Spot Instances can be terminated by AWS if the capacity is needed elsewhere.
      • Example:
        • A company running a genomic sequencing analysis, which is computation-intensive but can handle interruptions, might opt for Spot Instances to save on costs.

    • Reserved Instances:

      • Reserved Instances provide a significant discount (up to 72%) compared to On-Demand pricing in exchange for a one- or three-year commitment.
      • Use Cases:
        • Suitable for steady-state workloads with predictable usage patterns, such as a production web server or a database that is always on.
        • Ideal for businesses that can plan their capacity needs in advance.
      • Example:
        • An e-commerce company might use Reserved Instances for their website’s backend servers, which need to be available 24/7.

EC2 Instance Lifecycle

  • Lifecycle States:

    • Pending:
      • When an instance is first launched, it enters the pending state while AWS prepares the necessary resources.
    • Running:
      • Once the instance is ready, it enters the running state. At this point, the instance can be accessed, and users can start running applications on it.
      • Example:
        • A web server hosted on an EC2 instance can begin serving web pages once the instance is in the running state.
    • Stopping:
      • When an instance is being shut down, it enters the stopping state. During this state, the instance is in the process of stopping but has not yet completely stopped.
    • Stopped:
      • In the stopped state, the instance is shut down, but the data on the Amazon EBS (Elastic Block Store) volume is preserved. The instance can be restarted at any time.
      • Example:
        • A development environment can be stopped when not in use to save on costs, and then restarted when needed.
    • Terminating:
      • When an instance is permanently shut down, it enters the terminating state. Once terminated, the instance is deleted, and all associated resources are freed.

  • Operations:

    • Stop and Start (Amazon EBS-backed Instances Only):
      • Stopping and starting an instance moves it to a new host computer, but the instance retains its private IPv4 address. If the instance has an IPv6 address, it also retains that address.
      • Example:
        • If an application on an instance is not performing well, stopping and starting the instance might resolve the issue by moving the instance to a new host.
    • Hibernate (Amazon EBS-backed Instances Only):
      • When hibernating an instance, the operating system saves the contents of the instance’s memory (RAM) to the Amazon EBS root volume. This allows the instance to resume from the exact state it was in when it was hibernated.
      • Example:
        • Hibernation might be used for a long-running computational task that can be paused and resumed without starting from scratch.
    • Terminate:
      • Termination is the process of permanently shutting down an instance. Once terminated, the instance cannot be restarted, and all data stored on instance store volumes is lost.
      • Example:
        • Terminating an instance is useful when you no longer need the instance and want to stop incurring charges.

AWS Auto Scaling

  • Purpose:
    • AWS Auto Scaling ensures that the number of Amazon EC2 instances running your application adjusts automatically to maintain consistent, predictable performance at the lowest possible cost.
    • How It Works:
      • Auto Scaling monitors your applications and adjusts capacity by adding or removing EC2 instances based on the conditions you define. For example, you can set up scaling policies to add more instances when CPU utilization is high and remove instances when it’s low.
      • Example:
        • A news website might use Auto Scaling to automatically add instances during a high-traffic event (e.g., breaking news) and scale down during quieter times.

Elastic Load Balancing

  • Functionality:
    • Elastic Load Balancing automatically distributes incoming application traffic across multiple targets, such as EC2 instances, containers, and IP addresses, in one or more Availability Zones.
    • Key Features:
      • Scalability: By distributing traffic, the load balancer helps ensure that no single instance is overwhelmed, improving overall application performance.
      • Health Checks: The load balancer performs health checks on registered targets to ensure that only healthy targets receive traffic.
    • Example:
      • A global e-commerce site might use Elastic Load Balancing to distribute traffic across multiple servers in different regions, ensuring high availability and quick response times.

AWS Storage Services

Amazon S3 (Simple Storage Service)

  • Overview:

    • Amazon S3 is an object storage service that offers industry-leading scalability, data availability, security, and performance. It allows users to store and retrieve any amount of data from anywhere on the web.
    • Key Features:
      • Scalability: S3 can store a virtually unlimited amount of data.
      • Durability: S3 is designed for 99.999999999% (11 9s) durability.
      • Security: S3 provides features like access control, encryption, and integration with other AWS security services.

  • Use Cases:

    • Data Storage:
      • S3 is ideal for storing large amounts of data, whether for short-term or long-term storage.
      • Example:
        • A media company might store high-definition videos on S3 to ensure they are available for streaming on demand.
    • Backup and Recovery:
      • Many organizations use S3 for backing up critical data. S3’s durability ensures that data is safe and available for recovery when needed.

  • Example:

    • A financial institution might back up transaction logs to S3 to ensure they can recover the data in case of a disaster.

  • Hosting Static Websites:

    • S3 can host static websites, including HTML, CSS, and JavaScript files.
    • Example:
      • A personal blog or portfolio site can be hosted on S3, benefiting from low-latency access and cost-effectiveness.

  • Data Archiving:

    • S3 integrates with Amazon S3 Glacier, which provides a cost-effective solution for long-term data archiving.
    • Example:
      • A research organization might archive old project data on S3 Glacier, accessing it only when needed for historical analysis.

  • Big Data Analytics:

    • S3 serves as a data lake, storing large amounts of structured and unstructured data, and integrates with AWS analytics and machine learning services.
    • Example:
      • A data science team might store raw data on S3 and use AWS analytics tools to process and analyze the data for insights.

  • Key Concepts:

    • S3 Buckets:
      • A bucket is a container for storing objects in S3. Each bucket is globally unique and can store an unlimited number of objects.
      • Example:
        • A company might create separate buckets for different departments, such as one for marketing materials and another for financial documents.
    • S3 Objects:
      • Objects are the fundamental entities stored in S3, consisting of data and metadata. An object can be up to 5TB in size.
      • Example:
        • A video file stored on S3 as an object might include metadata like the file format, size, and date of upload.
    • Versioning:
      • S3’s versioning feature keeps a record of all versions of an object, preventing accidental overwrites or deletions.
      • Example:
        • A legal firm might enable versioning to keep track of all revisions to important documents, ensuring they can always revert to a previous version if needed.

  • S3 Storage Classes:

    • Standard:
      • Designed for frequently accessed data that needs to be highly available and durable.
      • Example:
        • An application that serves images and videos to users might use the Standard class to ensure quick access and high availability.
    • Standard Infrequent Access (Standard IA):
      • A cheaper storage class for infrequently accessed data, such as log files or data archives.
      • Example:
        • Archived records of past transactions that are only accessed occasionally might be stored in Standard IA to save costs.
    • Intelligent Tiering:
      • Automatically moves data between two access tiers (frequent and infrequent) based on changing access patterns.
      • Example:
        • Data that is frequently accessed for the first few months and then rarely accessed might benefit from Intelligent Tiering to optimize storage costs.
    • One Zone Infrequent Access (One Zone IA):
      • Stores data in a single Availability Zone, making it cheaper but less durable.
      • Example:
        • Non-essential data like temporary files or secondary backups might be stored in One Zone IA to minimize costs.
    • Reduced Redundancy Storage (RRS):
      • Offers lower durability (99.99%) and is no longer recommended by AWS for most use cases.
      • Example:
        • Non-critical data that can tolerate occasional loss, such as a cache of web pages, might have used RRS in the past, though AWS now recommends other storage classes.

  • S3 Lifecycle Policies:

    • Lifecycle Management:
      • S3 lifecycle policies allow users to manage their objects so that they are stored cost-effectively throughout their lifecycle.
      • Example:
        • A company might set a lifecycle policy to move older data to Glacier after 90 days and delete it after 365 days to save on storage costs.
    • Transition Actions:
      • Moving objects to different storage classes as they age.
      • Example:
        • Automatically moving infrequently accessed data to the Standard IA class after 30 days.
    • Expiration Actions:
      • Deleting objects after a specified period.
      • Example:
        • Deleting logs that are older than a year to comply with data retention policies.

Amazon EBS (Elastic Block Store)

  • Overview:

    • Amazon EBS is a block storage service designed for use with EC2 instances. It provides persistent, high-performance storage for applications that require frequent reads and writes to large volumes of data.
    • Key Features:
      • Persistence: EBS volumes remain available and retain data even after the EC2 instance is stopped or terminated.
      • Scalability: Volumes can be scaled up or down without disrupting applications.
      • Snapshots: EBS supports creating snapshots of volumes, which can be used for backup, replication, and disaster recovery.

  • Use Cases:

    • Database Storage:
      • EBS is ideal for databases like MySQL, Oracle, and MongoDB that require consistent, low-latency performance.
      • Example:
        • An online retail platform might use EBS to store its transaction data in a database, ensuring quick access and reliability.
    • Big Data Analytics:
      • EBS can be used for storing and processing large datasets in analytics applications.
      • Example:
        • A financial services company might use EBS volumes to store and analyze stock market data in real-time.
    • Backup and Recovery:
      • EBS snapshots provide a quick and efficient way to back up data stored on EBS volumes.
      • Example:
        • A healthcare organization might create daily snapshots of patient records stored on EBS volumes to ensure data recovery in case of a failure.

  • Encryption:

    • EBS volumes can be encrypted to protect data at rest, meeting regulatory compliance requirements.
    • Example:
      • A government agency might encrypt its EBS volumes to comply with data security regulations, ensuring that sensitive information is protected.

Amazon Glacier

  • Overview:

    • Amazon Glacier is a secure, durable, and extremely low-cost storage service for data archiving and long-term backup. It is designed for data that is infrequently accessed and can withstand retrieval times of several hours.
    • Key Features:
      • Cost-Effectiveness: Glacier provides significant savings over traditional storage for long-term archives.
      • Durability: Glacier is designed for 99.999999999% durability over a given year.
      • Retrieval Options: Glacier offers multiple retrieval options ranging from a few minutes to several hours.

  • Use Cases:

    • Data Archiving:
      • Glacier is ideal for storing archival data that is rarely accessed, such as legal records, medical records, or financial statements.
      • Example:
        • A law firm might archive old case files on Glacier, knowing they can retrieve them if needed for future reference.
    • Backup:
      • Glacier can be used for long-term backups that do not require immediate access.
      • Example:
        • An IT department might use Glacier to store backups of old systems or historical data that is only accessed in case of a disaster.

  • Scenarios for Using Glacier:

    • Long-Term Data Storage:
      • Glacier is particularly useful for data that needs to be stored for years but is rarely accessed, such as compliance records.
    • Alternative to Tape Libraries:
      • Organizations that previously used magnetic tape for backup can switch to Glacier for better durability and less maintenance.
      • Example:
        • A media company might move its tape-based video archives to Glacier to reduce storage costs and simplify access.

Amazon EFS (Elastic File System)

  • Overview:

    • Amazon EFS is a scalable, fully managed file storage service that can be used by multiple EC2 instances simultaneously. EFS is designed for high throughput and low latency, making it ideal for big data and analytics, media processing, and content management applications.
    • Key Features:
      • Shared File Storage: EFS allows multiple EC2 instances to access the same file system, ensuring data consistency and simplifying data sharing across applications.
      • Scalability: EFS automatically scales up and down as the amount of data stored increases or decreases, with no need for manual intervention.
      • Reliability: EFS is designed to be highly available and durable, with data automatically replicated across multiple Availability Zones.

  • Use Cases:

    • Shared File Storage:
      • EFS is ideal for applications where multiple EC2 instances need to access the same data, such as content management systems, development environments, and shared data repositories.
      • Example:
        • A media production company might use EFS to store video assets that need to be accessed by multiple editors working on different EC2 instances.
    • Big Data and Analytics:
      • EFS provides high-throughput storage for processing large datasets in parallel, making it suitable for big data applications.
      • Example:
        • A research institution might use EFS to store and analyze genomic data across a cluster of EC2 instances.
    • Serverless Applications:
      • EFS integrates with AWS Lambda and other serverless services, providing persistent storage for serverless applications.
      • Example:
        • A web application built on AWS Lambda might use EFS to store user-uploaded files that need to be accessed across multiple Lambda functions.

  • When to Choose EFS:

    • Shared Data Access: Choose EFS when multiple EC

2 instances need to access the same file system.

  • Scalability: Use EFS when you expect the amount of data to grow unpredictably, as it automatically adjusts capacity based on demand.
  • Simplified Data Sharing: EFS is ideal for applications where data needs to be shared across different environments or services.
